# Generator 基础

function* gen() {
  return "first generator";
}

let generatorResult = gen();

console.log(generatorResult);

console.log(generatorResult.next()); //{value: "first generator", done: true}
console.log(generatorResult.next().value); //'first'

# Generator 注意事项

不能无限使用 next 中取值

//继续上面的例子
在执行一遍;
console.log(generatorResult.next().value); //undefined

因为Generator如同序列,一旦序列中的值被消费,你就不能再次消费它.所以第二次则返回undefined

# yeid 关键字

function* gen1() {
  yield "first", yield "second", yield "third";
}

let generatorResult1 = gen1();

console.log(generatorResult1.next().value); // first
console.log(generatorResult1.next().value); // second

yield使 Generator 函数暂停执行并将结果返回给调用者.因此第一次调用函数看到了yield后面的值 first,第二次看到 second,当下次调用generatorResult1的时候,从中断的地方恢复执行,由于它第一次暂停在此行yield 'first',. 所有带 yield 的 Generator 函数都会惰性求值的顺序执行

什么是惰性求值? 代码直到调用的时候才会执行

# done 属性

用户如何知道停止调用 next 函数呢? 每一次调用 next()方法后返回一个对象

{value:"value",done:false}

当我们消费了所有序列中的值后返回

{value:undefined,done:true}

# 向 Generator 传递函数

function* gen3() {
  var firstName = yield;
  var secondName = yield;
  console.log(firstName + secondName);
}

let fullName = gen3();

fullName.next();
fullName.next("anto");
fullName.next("aravinth"); //antoaravinth
  1. 第一次没有传值返回 undefined
  2. 第二次传值返回 anto
  3. 第三次传值赋值给 secondName.继续执行后面语句

# 使用 Generator 处理异步调用

let gengerator;
let getDataOne = () => {
  setTimeout(function() {
    gengerator.next("dumy data one");
  }, 1000);
};
let getDataTwo = () => {
  setTimeout(function() {
    gengerator.next("dumy data two");
  }, 1000);
};

function* main() {
  let dataOne = yield getDataOne();
  let dataTwo = yield getDataTwo();
  console.log("data one", dataOne);
  console.log("data two", dataTwo);
}

gengerator = main();
gengerator.next();

与期望的一样,main 函数看上去像同步调用一样.但 2 各调用都是异步的.这些调用不会阻塞代码并以异步方式运行

注意:虽然 yield 使语句暂停,但它不会让调用者等待,也就是说调用者不会阻塞.

gengerator.next();
//后面加上
console.log("will be  print"); //会先执行
