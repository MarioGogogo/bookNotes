# NodeJs

# NodeJs是什么

Node.js是一个基于 Chrome V8 引擎的JavaScript运行环境(runtime)。

虽然node是用javascript的语法，但是并非是完全的javascript，我们知道javascript是包含了ECMAScript、DOM、BOM，而node则不包含DOM和BOM，但是它也提供了一系列模块供我们使用，如http、fs模块。

# 特性

node是使用了事件驱动、非阻塞式 I/O的模型

事件驱动 发送事件后，通过回调的消息通知机制通知
非阻塞式 I/O 如操作文件，通过非阻塞异步的方式读取文件

# 进程和线程

###进程

进程是操作系统分配资源和调度任务的基本单位,线程是建立在进程上的一次程序运行单位，一个进程上可以有多个线程。

=====

多进程：启动多个进程，可以一块执行多个任务。

=====

线程：进程内一个相对独立，可调用的执行单元，与同属一个进程的线程共享进程的资源。

=====

多线程：进程内多个线程，多个线程可以一块执行多个任务。

=====

首先单线程只针对主进程，I/O操作底层多线程调度，js单线程指的是js的主线程是单线程。

单线程并不是单进程，cpu有几个核就可以启几个进程，不浪费cpu能力

适合i/o操作密集 不适合cpu操作密集

# 什么是同步与异步

Math.sqrt(2);
console.log('Hi');

第一个函数返回时，就拿到了预期的返回值：2的平方根。

第二个函数返回时，就看到了预期的效果：在控制台打印了一个字符串。

所以这两个函数都是同步的。

异步：如果在函数A返回的时候，调用者还不能够得到预期结果，而是需要在将来通过一定的手段得到，那么这个函数就是异步的。

fs.readFile('foo.txt', 'utf8', function(err, data) {
    console.log(data);
});

# 执行顺序

const { readFile } = require("fs");
const EventEmitter = require("events");

class EE extends EventEmitter {}

const yy = new EE();

yy.on("event", () => {
  console.log("我写在第一行");
});

setTimeout(() => {
  console.log("0 毫秒后到期执行定时器回调");
}, 0);

setTimeout(() => {
  console.log("100 毫秒后到期执行定时器回调");
}, 100);

readFile("./package.json", "utf-8", data => {
  console.log("完成文件 1的回调");
});

setImmediate(() => {
  console.log('setImmediate的回调');
});

readFile("./README.md", "utf-8", data => {
  console.log("完成文件 2的回调");
});

// 优先执行
process.nextTick(()=>{
  console.log('nextTick1111的回调');
})

Promise.resolve()
.then(()=>{
  yy.emit('event')
  process.nextTick(()=>{
    console.log('nextTick22222的回调');
  })
  console.log('Promise1111的回调');
}
).then(()=>{
  console.log('Promise22222的回调');
})

// nextTick1111的回调
// 我写在第一行
// Promise1111的回调
// Promise22222的回调
// nextTick22222的回调
// 0 毫秒后到期执行定时器回调
// setImmediate的回调
// 完成文件 1的回调
// 完成文件 2的回调
// 100 毫秒后到期执行定时器回调
