    # http协议

    # http协议的特点

    简单快速：每一个url都是固定的 处理起来方便 灵活 通过一个http协议，完成不同数据类型的传输 无连接 只连接一次就会断掉 无状态 客户端 和服务端是2种状态，一次连接之后断了，下次连接服务端是无法区分你是来自哪个同一个人同一个状态（可以通过session做到记住状态） http报文的组成部分

    # 请求报文

    请求行 http方法 页面地址 http协议 版本 请求头 key&&value的值来告诉服务端我要哪些内容 空行 作用告诉服务端解析请求头告一段落下面是请求体 请求体

    # 响应报文

    状态行 响应头 空行 响应体 比如 浏览器向服务器请求要图片：输入url地址就是发送请求报文，服务端要响应就发送响应报文； http的方法

    # post与get的区别

    1. get在浏览器回退是无害的，而post会再次发出请求
    2. get产生url地址可被收藏，而post不可以
    3. get请求被浏览器主动缓存，而post不会，除非手动设置
    4. get请求只能url编码，而post支持多种编码方式
    5. get请求参数会被完整保留在浏览器历史里，而post的参数不会
    6. get请求url传送有长度限制()，而post没有限制
    7. 对参数的数据类型，get只接受ASCII字符，而post没有限制
    8. get比post更不安全，因为参数暴露在url，不能用来传敏感信息
    9. get参数通过rul传递，post放在request body中

    # http状态码

    1xx:指示信息-请求已被接受继续处理 2xx:成功-表示请求被成功接受 3xx:重定向-要完成请求必须进行更进一步操作 4xx:客户端错误，请求有语法错误或请求无法实现 5xx:服务端错误，服务器未能实现合法的请求

    # 什么是持久链接

    http协议采用“请求-应答”模式，在非keep-alive模式下客户端与服务器要新建一次连接，完成后即可断开连接 开启keep-alive模式

    # 什么是管线化

    普通连接消息传递 请求1->响应1->请求2->响应2 某个连接上的消息变成这样 请求1->请求2->响应1->响应2 http缓存
